# Афіша подій Житомир 2026 — Next.js + Prisma + PostgreSQL

Навчальний вебзастосунок-афіша подій міста Житомир. Функції: каталог подій з фільтрами та пошуком, календар як віджет на головній, сторінка події з квитками, замовлення, модуль відгуків з модерацією та розмежування доступу за ролями USER/ORGANIZER/ADMIN.

## Швидкий старт (локально)

### 1) Вимоги
- Node.js 18.17+ (краще 20+)
- Docker Desktop (для Postgres) або встановлений Postgres локально

### 2) Запуск БД (Docker)
У корені проєкту:
```bash
docker compose up -d
```

### 3) Налаштування .env
Скопіюй приклад:
```bash
cp .env.example .env
```

### 4) Встановити залежності
```bash
npm install
```

### 5) Міграції та seed (тестові дані)
```bash
npx prisma migrate dev -n init
npx prisma db seed
```

### 6) Запуск застосунку
```bash
npm run dev
```

Відкрити: http://localhost:3000

## Команди
- `npm run lint` — лінтинг
- `npm run test` — юніт-тести (Vitest)
- `npm run build` — збірка

## Сторінки
- `/` — каталог подій + календар-віджет
- `/events/[id]` — сторінка події (квитки, відгуки, обране)
- `/organizers/[id]` — сторінка організатора з його опублікованими подіями
- `/login`, `/register` — вхід/реєстрація
- `/account` — кабінет користувача
  - `/account/orders` — історія покупок
  - `/account/reviews` — мої відгуки (зі статусами модерації)
  - `/account/favorites` — обране
- `/panel` — панель організатора (ORGANIZER/ADMIN)
  - `/panel/events` — події (CRUD, статуси, квитки, замовлення)
  - `/panel/orders` — замовлення по подіях
- `/admin` — адмін-панель (ADMIN)
  - `/admin/users` — керування ролями
  - `/admin/reviews` — модерація відгуків
  - `/admin/events` — контроль контенту (статуси подій)

## Ролі
- USER: перегляд каталогу, оформлення замовлень, створення відгуків, історія покупок
- ORGANIZER: створення/редагування подій, керування квитками, перегляд замовлень по своїх подіях
- ADMIN: керування ролями, модерація відгуків, контроль статусів подій

## Примітка про "купівлю квитків"
У прототипі оплата не інтегрована. Натискання "Оформити замовлення" створює запис Order у БД та зменшує доступну кількість квитків.
